# Шина хранения

<GameScene zoom="8" background="transparent">
<ImportStructure src="../assets/blocks/storage_bus.snbt" />
</GameScene>

Захотелось *сохранить* своего монстра-сундука вместо того, чтобы заменять его на что-то разумное? Представляем шину хранения!

Шина хранения превращает инвентарь, к которому она прикасается, в [сетевое хранилище](../ae2-mechanics/import-export-storage.md). Она делает это, позволяя сети видеть содержимое этого инвентаря, и помещая предметы в этот инвентарь или извлекая их из него для выполнения задач [устройств](../ae2-mechanics/devices.md), которые помещают предметы в сетевое хранилище или извлекают их оттуда.

Исходя из философии AE2, заключающейся в появлении новых механик благодаря взаимодействию функций [устройств](../ae2-mechanics/devices.md), вам не обязательно использовать шину хранения *для хранения*. Используя [подсети](../ae2-mechanics/subnetworks.md), вы можете сделать одну шину хранения (или несколько шин хранения) *единственным* хранилищем в сети и использовать её в качестве источника или пункта назначения для передачи предметов. (см. ["подсеть-труба"](../example-setups/pipe-subnet.md))

ВАЖНО: Большие оптимизированные инвентари, такие как ящики, в порядке, но большие *неоптимизированные* инвентари со множеством слотов, например, гигантские сундуки, ужасны для производительности при использовании с шинами хранения.

Они являются [частями кабеля](../ae2-mechanics/cable-subparts.md).

## Фильтрация

По умолчанию шина будет хранить всё. Предметы, помещённые в её слоты фильтра, будут действовать как белый список, разрешая хранение только этих конкретных предметов.

Предметы и жидкости можно перетаскивать в сл��ты из JEI/REI, даже если у вас на самом деле нет ни одного такого предмета.

Щёлкните правой кнопкой мыши по контейнеру с жидкостью (например, ведру или резервуару), чтобы установить эту жидкость в качестве фильтра вместо предмета ведра или резервуара.

## Приоритет

Приоритеты можно установить, щёлкнув по ключу в правом верхнем углу графического интерфейса. Предметы, поступающие в сеть, будут начинать с хранилища с наивысшим приоритетом в качестве своей первой цели. В случае, если у двух хранилищ одинаковый приоритет, если одно из них уже содержит предмет, они предпочтут это хранилище любому другому. Любые отфильтрованные хранилища будут считаться уже содержащими предмет, когда находятся в одной группе приоритетов с другими хранилищами. Предметы, извлекаемые из хранилища, будут извлекаться из хранилища с наименьшим приоритетом. Эта система приоритетов означает, что по мере помещения и извлечения предметов из сетевого хранилища, хранилища с более высоким приоритетом будут заполняться, а хранилища с более низким приоритетом будут опустошаться.

## Настройки

*   Шину можно разделить (отфильтровать) по тому, что в данный момент находится в соседнем инвентаре
*   Можно запретить или разрешить сети видеть предметы в соседнем инвентаре, которые шина не может извлекать (например, шина хранения не может извлекать предметы из среднего слота ввода <ItemLink id="inscriber" />)
*   Шина может фильтровать как при вставке, так и при извлечении, или только при вставке
*   Шина может быть двунаправленной, только для вставки или только для извлечения

## Улучшения

Шина хранения поддерживает следующие [улучшения](upgrade_cards.md):

*   <ItemLink id="capacity_card" /> увеличивает количество слотов фильтра
*   <ItemLink id="fuzzy_card" /> позволяет шине фильтровать по уровню прочности и/или игнорировать NBT предмета
*   <ItemLink id="inverter_card" /> переключает фильтр с белого списка на чёрный список
*   <ItemLink id="void_card" /> уничтожает вставленные предметы, если прикреплённый инвентарь полон, что полезно для предотвращения заторов на фермах. Будьте осторожны и используйте разделение!
