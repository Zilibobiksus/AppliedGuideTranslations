# МЭ накопитель

<GameScene zoom="8" background="transparent">
  <ImportStructure src="../assets/blocks/drive.snbt" />
</GameScene>

Накопитель — это [устройство](../ae2-mechanics/devices.md), в которое вы вставляете [ячейки хранения](storage_cells.md), чтобы использовать их для [сетевого хранилища](../ae2-mechanics/import-export-storage.md). У него есть 10 слотов, каждый из которых принимает ячейку.

Если по какой-то причине вы хотите, вы можете вталкивать и выталкивать ячейки из его инвентаря с помощью любой логистики предметов, например, желобов или шин ae2.

Его можно поворач��вать с помощью <ItemLink id="certus_quartz_wrench" />.

## Светодиоды состояния ячейки

Ячейки в накопителе имеют светодиод, который показывает их состояние:

| Цвет  | Состояние                                                                           |
| :----- | :------------------------------------------------------------------------------- |
| Зелёный  | Пусто                                                                            |
| Синий   | Есть содержимое                                                                |
| Оранжевый | [Виды](../ae2-mechanics/bytes-and-types.md) заполнены, нельзя добавить новые виды     |
| Красный    | [Байты](../ae2-mechanics/bytes-and-types.md) заполнены, больше нельзя вставить предметы |
| Чёрный  | Нет энергии или у накопителя нет [канала](../ae2-mechanics/channels.md)                 |

## Приоритет

Приоритеты можно установить, щёлкнув по гаечному ключу в правом верхнем углу графического интерфейса. Предметы, поступающие в сеть, будут начинать с хранилища с наивысшим приоритетом в качестве своей первой цели. В случае, если два хранилища или ячейки имеют одинаковый приоритет, если одна из них уже содержит предмет, они предпочтут это хранилище любому другому. Любые [разделённые](cell_workbench.md) ячейки будут считаться уже содержащими предмет, когда находятся в одной группе приоритета с другими хранилищами. Предметы, удаляемые из хранилища, будут удаляться из хранилища с наименьшим приоритетом. Эта система приоритетов означает, что по мере вставки и извлечения предметов из сетевого хранилища хранилища с более высоким приоритетом будут заполняться, а хранилища с более низким приоритетом будут опустошаться.
