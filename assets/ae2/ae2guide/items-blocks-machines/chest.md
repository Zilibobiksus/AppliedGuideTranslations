# МЭ сундук

<GameScene zoom="8" background="transparent">
<ImportStructure src="../assets/blocks/chest.snbt" />
</GameScene>

МЭ сундук действует как миниатюрная сеть с <ItemLink id="terminal" />, <ItemLink id="drive" /> и <ItemLink id="energy_acceptor" />. Хотя он может использоваться в качестве крошечной сетевой системы хранения, его ёмкость для всего лишь одной [ячейки хранения](../items-blocks-machines/storage_cells.md) означает, что его полезность в этом качестве ограничена.

Вместо этого он полезен для взаимодействия именно с ячейкой хранения, установленной внутри него. Встроенный терминал может видеть и получать доступ только к предметам в установленном накопителе, в то время как [устройства](../ae2-mechanics/devices.md) в основной сети могут получать доступ к предметам в любом [сетевом хранилище](../ae2-mechanics/import-export-storage.md), включая МЭ сундуки.

Он имеет 2 разных графических интерфейса и имеет стороны для транспортировки предметов. Взаимодействие с верхним терминалом открывает встроенный терминал. Предметы можно вставлять в установленную ячейку хранения только через эту грань, но извлечь их нельзя. Взаимодействие с любой другой гранью открывает графический интерфейс со слотом для ячейки хранения и настройками приоритета. Ячейку можно вставлять и извлекать с помощью логистики предметов только через грань со слотом ячейки.

Его можно поворачивать с помощью <ItemLink id="certus_quartz_wrench" />.

Он имеет небольшой буфер хранения энергии AE, поэтому если он не подключён к сети с [энергохранилищем](../items-blocks-machines/energy_cells.md), вставка или извлечение слишком большого количества предметов за раз может привести к его отключению.

Терминал можно окрасить с помощью <ItemLink id="color_applicator" />.

## Настройки

МЭ сундук имеет все те же настройки, что и <ItemLink id="terminal" /> или <ItemLink id="crafting_terminal" />. Однако он не поддерживает <ItemLink id="view_cell" />.

## Светодиоды состояния ячейки

Ячейки в сундуке имеют светодиод, который показывает их состояние:

| Цвет  | Состояние                                                                           |
| :----- | :------------------------------------------------------------------------------- |
| Зелёный  | Пусто                                                                            |
| Синий   | Есть содержимое                                                                |
| Оранжевый | [Виды](../ae2-mechanics/bytes-and-types.md) заполнены, нельзя добавить новые виды     |
| Красный    | [Байты](../ae2-mechanics/bytes-and-types.md) заполнены, больше нельзя вставить предметы |
| Чёрный  | Нет энергии или у накопителя нет [канала](../ae2-mechanics/channels.md)                 |

## Приоритет

Приоритеты можно установить, щёлкнув по гаечному ключу в правом верхнем углу графического интерфейса слота ячейки. Предметы, поступающие в сеть, будут начинать с хранилища с наивысшим приоритетом в качестве своей первой цели. В случае, если два хранилища или ячейки имеют одинаковый приоритет, если одна из них уже содержит предмет, они предпочтут это хранилище любому другому. Любые [разделённые](cell_workbench.md) ячейки будут считаться уже содержащими предмет, когда находятся в одной группе приоритета с другими хранилищами. Предметы, удаляемые из хранилища, будут удаляться из хранилища с наименьшим приоритетом. Эта система приоритетов означает, что по мере вставки и извлечения предметов из сетевого хранилища хранилища с более высоким приоритетом будут заполняться, а хранилища с более низким приоритетом будут опустошаться.
